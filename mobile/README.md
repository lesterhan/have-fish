# have-fish mobile

Android client for have-fish ‚Äî React Native + Expo (SDK 52, Expo Router).
Primarily the Fish Pie group-expense flow on the go.

## Quick start

```bash
bun install
bun run start        # Metro bundler ‚Äî scan the QR with Expo Go on your phone
```

Expo Go is the fastest loop: it hot-reloads JS and every native module this app
uses (`expo-secure-store`, `expo-haptics`, router) runs inside it, so no local
Android toolchain is needed for day-to-day work.

The app talks to a have-fish backend. The **server URL is entered in the app**
(stored in SecureStore via `lib/auth.ts`), not hardcoded ‚Äî point it at your dev
backend's LAN address (e.g. `http://192.168.x.x:8887`), not `localhost`, since
that resolves to the phone itself.

## Native dev build (optional)

Only needed when Expo Go can't represent a change (e.g. a new native module).
Requires a JDK and the Android SDK on PATH, plus an emulator or USB device:

```bash
export ANDROID_HOME=$HOME/Android/Sdk
export PATH="$ANDROID_HOME/platform-tools:$PATH"
bun run android      # expo run:android (runs expo prebuild under the hood)
```

`android/` is generated by `expo prebuild` and is **not** committed ‚Äî never
hand-edit it; changes belong in `app.json` or a config plugin under `plugins/`.

## Dependency versions

Native deps are pinned to the versions Expo SDK 52 expects ‚Äî run
`npx expo install --check` after touching `package.json`. Mismatched native
modules surface as Metro/codegen errors at build time, not install time.

## Releases

Signed APKs are built in CI (`.github/workflows/build-android.yml`) and
published as GitHub Releases for install via Obtainium. The build, signing, and
keystore setup are documented in the epic at
`planning/epics/mobile-local-apk-ci.md`.
